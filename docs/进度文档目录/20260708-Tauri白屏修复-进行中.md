# Tauri桌面应用白屏问题修复

## 创建时间
2026-07-08

## 问题描述
Tauri桌面应用启动后窗口白屏，无法显示任何内容。

## 问题分析

### 根因定位
通过Playwright MCP工具调试发现：
1. 页面HTML加载正常（约30KB）
2. React根节点为空：`<div hidden=""><!--$--><!--/$--></div>`（Suspense空占位符）
3. Next.js RSC payload为空数组：`window.__next_f.length === 0`
4. `document.body.innerText.length === 0`

### 根本原因
`src/app/page.tsx`存在bug：
- 当`isAuthenticated`为false时（默认状态），`return null`
- 导致未认证用户看到白屏

同时存在路由冲突：
- `src/app/page.tsx` 映射到 `/`
- `src/app/(dashboard)/page.tsx` 也映射到 `/`（路由组无URL前缀）
- 两者冲突，Next.js优先使用非路由组的`src/app/page.tsx`
- 导致`src/app/(dashboard)/layout.tsx`的认证逻辑（未认证显示LoginForm）无法生效

## 修复方案

### 修改内容
1. 删除`src/app/page.tsx`
2. 让`src/app/(dashboard)/page.tsx` + `src/app/(dashboard)/layout.tsx`接管`/`路径
3. layout.tsx已处理认证逻辑：未认证时显示LoginForm，认证后展示完整Dashboard

### 验证结果
- Playwright访问`http://localhost:3002/`
- 页面正常显示登录表单：
  - "智能文档知识库"标题
  - "文档管理 · 相册管理"副标题
  - 邮箱/密码输入框
  - 登录按钮
  - "还没有账号？立即注册"链接
  - "按 ? 查看快捷键"提示
- `forms: 1, inputs: 2, cards: 2` - UI元素正常渲染
