### 4. Rust 编译错误修复（48个错误→0个）
- **E0255（命令宏重复定义）**：重构为 Tauri v2 标准结构
  - lib.rs 添加 `pub fn run()` 函数，包含 `generate_handler!`
  - main.rs 简化为只调用 `knowledge_base::run()`
  - 所有 `#[command]` 改为 `#[tauri::command]` 全路径
  - 命令函数用 `fn`（非 pub），避免 `__cmd__*` 宏重复 import
- **E0583（模块路径）**：Cargo.toml 添加 `[lib] name = "knowledge_base" path = "src/lib.rs"`
- **E0432（TrayEvent）**：tray.rs 改用 `TrayIconEvent`（Tauri v2 新名称）
- **E0599（Manager trait）**：tray.rs 添加 `use tauri::Manager;`
- **E0277（str 类型 Sized）**：lib.rs:545 明确指定 `key_points: Option<Vec<String>>`
- **E0433（time crate）**：Cargo.toml 添加 `time = "0.3"`
- **E0382（borrow moved）**：`version.file_name` 改为 `.clone()`
- **E0597（生命周期）**：`Vec<&dyn ToSql>` 改为 `Vec<Box<dyn ToSql>>`
- **E0277（Tauri error）**：DbError 添加 `Tauri(#[from] tauri::Error)` variant

### 5. 链接器错误修复（LNK2019）
- 根因：用户系统全局 `CC=gcc`，bundled sqlite3 用 gcc 编译，但 Rust MSVC target 用 link.exe 链接，符号不兼容
- 解决：创建 `src-tauri/build-msvc.ps1` 脚本，调用 VS DevShell 设置 MSVC 环境后编译
- 创建 `src-tauri/.cargo/config.toml` 持久化 `CC=cl.exe` 设置
- 创建 `src-tauri/build-msvc.bat` 备用脚本
